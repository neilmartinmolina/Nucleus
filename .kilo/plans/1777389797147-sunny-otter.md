# Nucleus Feature Audit & Implementation Plan

## Navigation Flow (Confirmed)

**Sidebar menu order:**
1. Dashboard (stats + recent activity)
2. Folders (grid of folders, create new)
3. Websites (table of all websites, create/edit/delete)
4. Users (user management, roles, permissions) — admin only

### Phase 2 — Convert `view-folder.php` to card UI (light/navy theme, index.php style)

**Base style:** Light theme matching `index.php`:
- Background: white (`bg-white`)
- Accent colors: navy (`#043873`), CTA blue (`#4F9CF9`), accent yellow (`#FFE492`)
- Cards: white with slate border (`bg-white`, `border-slate-200`, rounded-xl)
- Typography: slate-800 headings, slate-500 body; use Tailwind utility classes

**Structure (from `testviewfolder.php` adapted):**
- Keep PHP auth (`view_projects` permission) + folder query + websites query
- Stats row (3 cards): Total projects, Updated this week, Needs update
  - Queries: `COUNT(*)`, weekly count (`WHERE lastUpdatedAt >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)`), needs-update count (`WHERE status IN ('updating','issue') OR lastUpdatedAt < DATE_SUB(CURDATE(), INTERVAL 15 DAY)`)
- Search input + Status filter dropdown + Sort dropdown
- Project grid (responsive: 1 col → 2 cols → 3 cols)
- Breadcrumb: `Folders / <folder name>` (links to `dashboard.php?page=folders`)
- "Add project" button → opens inline modal (not navigate)

**Project card:**
- Name (h2), URL (small, text-slate-500)
- Status badge (colored pill)
- "Last updated X ago" (computed from `lastUpdatedAt`)
- "Mark updated now" button → click POSTs to `handlers/update_website.php` with `{csrf_token, websiteId, status: "updated"}`; on success, updates version and card in place

**Status computation:**
- From `lastUpdatedAt` → days diff
  - ≤14 days: `Up to date` (green)
  - 15–29 days: `Needs update` (yellow)
  - ≥30 days: `30d+ old` (red)
- Server-side: embed a `data-status` attribute on each card for filter
- Client-side: JS filters by status value

**Add project modal:**
- Inline modal in `view-folder.php` with form fields (website name, URL, version default 1.0.0)
- POST to `handlers/create_website.php` (needs to exist — currently websites create is via `websites_content.php` POST; we can either reuse that endpoint or create a new one that accepts AJAX)
- On success: close modal, prepend new card to grid, refresh stats

**Mark updated button implementation:**
- Attach click handler to `.mark-updated` buttons
- Send POST (JSON) to `handlers/update_website.php` with CSRF token and websiteId
- Handler auto-increments version and sets status to "updated"
- On success: update the card's displayed version, lastUpdatedAt (to "Just now"), status badge color

**Badge CSS (add to page `<style>`):**
```css
.badge-updated { background: #d1fae5; color: #065f46; }
.badge-updating { background: #fef3c7; color: #92400e; }
.badge-issue { background: #fee2e2; color: #991b1b; }
```
*(Light pastel backgrounds, dark text for contrast on white cards)*

**Empty folder handling:**
- Stats row still visible (counts zero or N/A as appropriate)
- Search/filter visible
- Grid shows "No projects in this folder yet" message with "Add project" CTA

## Routing Architecture

**Landing page:** `index.php` — marketing page, unauthenticated only; authenticated users auto-redirect to `dashboard.php`

**Main app:** `dashboard.php` — dashboard layout with sidebar + top nav; loads page content via AJAX into `#pageContent` from `get_content.php?tab=<page>`

**Content endpoints** (`get_content.php` cases):
- `?tab=dashboard` → `dashboard_content.php`
- `?tab=websites` → `websites_content.php`  
- `?tab=folders` → `folders_content.php`
- `?tab=view-folder` → `view-folder.php` (special: requires `folderId` GET param)
- `?tab=usermanagement` → `usermanagement_content.php`
- `?tab=activity` → `activity_log.php` (new)

**Direct PHP pages:**
- `view-folder.php` — renders full page (not loaded via get_content currently); will eventually be integrated into dashboard layout or kept standalone with sidebar include

**All internal redirects** in content handlers must use `dashboard.php?page=<tab>` (e.g., after create/delete actions).
